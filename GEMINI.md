# プロジェクト概要

このプロジェクトは、注文データをチェックおよび処理するために設計されたPythonアプリケーションです。SQL Serverデータベースから注文情報を取得し、さまざまなチェックとデータ操作を実行した後、結果を要約したExcelレポートを生成します。

## 主要機能

アプリケーションのワークフローは次のとおりです。

1.  **ユーザー入力:** ユーザーは注文日とユーザーIDの入力を求められます。
2.  **データ取得:** アプリケーションは、SQL Serverデータベースから注文データと製品データをフェッチします。
3.  **データ処理:**
    *   日付がフォーマットされ、リードタイムが計算されます。
    *   出荷日と納期は、休日やその他のビジネスルールに対して検証されます。
    *   輸出対象製品のチェックが行われます。
    *   測定単位が変換されます（例：缶からキログラムへ）。
    *   潜在的な問題や警告を強調する最終的な「チェック」列がデータに追加されます。
4.  **レポート生成:** 処理されたデータは、フォーマットされたExcelスプレッドシート（`result.xlsx`）にまとめられます。アプリケーションは、同じデータを含むCSVファイル（`test.csv`）とテキストファイル（`df_order_grid.txt`）も生成します。

## 主要技術

*   **Python:** アプリケーションの主要なプログラミング言語。
*   **Pandas:** データ操作と分析に使用されます。
*   **SQL Server:** 注文および製品情報を保存するために使用されるデータベース。
*   **XlsxWriter:** 最終的なExcelレポートの作成とフォーマットに使用されます。

# ビルドと実行

このプロジェクトには正式なビルドプロセスはありません。コマンドラインから直接実行するように設計されています。

## アプリケーションの実行

アプリケーションを実行するには、`main.py`スクリプトを実行します。

```bash
python main.py
```

その後、アプリケーションは注文日とユーザーIDの入力を求めます。

## 依存関係

このプロジェクトはいくつかのPythonライブラリに依存しています。これらはpipを使用してインストールできます。

```bash
pip install pandas xlsxwriter
```

**注:** `pyodbc`など、SQL Serverに接続するためのドライバーも必要になります。

# 開発規約

*   **モジュラー設計:** コードは、それぞれが特定の責任を持ついくつかのモジュールに整理されています（例：データ取得用の`select_data.py`、ユーザーインタラクション用の`user_interface.py`）。
*   **型ヒント:** コードは、可読性と保守性を向上させるために型ヒントを使用しています。
*   **テスト:** このプロジェクトにはテスト目的のモックデータクラス（`SelectOrder_Test`）が含まれていますが、専用のテストフレームワークはありません。
*   **構成:** データベース接続の詳細は、おそらく別の追跡されていないファイル（例：`sql_server.py`）に保存されています。

## 尾頭追記
- `sql_server.py`は`<各PCのpath>/site-packages`に保存されています。
